## Catatan Pembelajaran
[Tulis 3-5 poin pembelajaran penting yang Anda dapatkan dari handson ini]
1. Interface Segregation Principle (ISP) mengajarkan bahwa antarmuka tidak boleh terlalu besar atau memuat terlalu banyak fungsi. Sebaliknya, interface harus kecil, fokus, dan relevan dengan kebutuhan klien sehingga kelas tidak dipaksa mengimplementasikan metode yang tidak dibutuhkan.

2. Dependency Inversion Principle (DIP) menekankan bahwa modul level tinggi tidak boleh bergantung pada modul level rendah. Keduanya harus bergantung pada abstraction (interface). Dengan cara ini, sistem menjadi lebih fleksibel, mudah diuji, dan mudah diekstensikan.

3. Dalam penerapan DIP, saya memahami pentingnya Dependency Injection untuk memberikan dependency dari luar, bukan membuatnya langsung di dalam class. Hal ini membuat class lebih loosely-coupled dan mendukung penggantian implementasi dengan mudah.
